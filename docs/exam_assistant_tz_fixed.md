## Требования

1. Обеспечить асинхронность доступа для одновременной работы нескольких пользователей.  
2. Поддержка нескольких групп в день с маршрутизацией данных.  
3. Поддержка 3 языков: русский, казахский, английский.  
4. Оценка должна возвращаться на русском и языке ответа.  
5. Упростить взаимодействие — убрать повторные уточнения.  
6. Исключить потерю данных при записи результатов.  
7. Управление несколькими ботами и наборами вопросов.  
8. Поддержка аренды ботов с оплатой и доступом.

## Этапы разработки

1. Основной функционал

## Архитектура

- Telegram-бот (frontend)  
- Backend (Python сервер)  
- База вопросов (Google Sheets)  
- Whisper (распознавание речи)  
- ChatGPT (оценка)  

---

## Сценарий работы

1. Пользователь выбирает билет  
2. Получает доступ к боту  
3. Вводит данные (ФИО, дисциплина и т.д.)  
4. Озвучивает ответ  
5. Система:
   - транскрибирует
   - переводит
   - оценивает
