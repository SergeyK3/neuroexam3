## Требования

1. Обеспечить асинхронность доступа для одновременной работы нескольких пользователей.  
2. Поддержка нескольких групп в день с маршрутизацией данных.  
3. Поддержка 3 языков: русский, казахский, английский.  
4. Ответ бота должен учитывать язык последнего сообщения студента; для голосового ответа без нового текста используется язык сессии.  
5. Упростить взаимодействие — убрать повторные уточнения.  
6. Исключить потерю данных при записи результатов.  
7. Управление несколькими ботами и наборами вопросов.  
8. Поддержка аренды ботов с оплатой и доступом.
9. Поддержка команд `/start` и `/new` для начала новой попытки.
10. Запрет на подсказки, раскрытие эталонов и ключевых элементов ответа.
11. Обязательный дисклеймер в сообщении с оценкой:
   - это предварительная оценка;
   - бот работает в тестовом режиме;
   - окончательную оценку выставляет преподаватель.

## Сценарий работы

1. Пользователь запускает `/start` или `/new`.  
2. Бот отправляет приветственное сообщение:
   - успокоиться;
   - сделать несколько глубоких вдохов и выдохов;
   - не торопиться;
   - внимательно читать вопросы.
3. Пользователь выбирает язык.  
4. Пользователь вводит регистрационные данные (дисциплина, вид контроля, группа, ФИО).  
5. Пользователь отправляет ответы по ключам билета текстом или голосом.  
6. Одно голосовое сообщение может содержать ответы сразу на несколько вопросов.  
5. Система:
   - транскрибирует
   - сегментирует единый ответ по `question_key`
   - оценивает каждый фрагмент отдельно
   - считает среднюю предварительную оценку по билету
   - сохраняет результат
   - возвращает оценку с обязательным дисклеймером

## Оценка

Основной режим:
- coverage / rubric по смысловым элементам  

Резервный режим:
- semantic similarity по эмбеддингам  

Правила:
- оценка должна учитывать полноту и смысл ответа, а не только буквальное совпадение;
- частичное раскрытие элемента учитывается как частичное покрытие;
- орфография, порядок изложения и нерелевантная "вода" не должны быть самостоятельным основанием для снижения оценки, если смысл сохранён;
- студенту нельзя раскрывать эталон, ключевые элементы эталона или правильную формулировку ответа.
