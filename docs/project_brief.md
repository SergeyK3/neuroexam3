# Технический бриф: Python-нейроэкзаменатор

## 1. Цель проекта

Разработать Python-сервис нейроэкзаменатора, который заменяет текущий n8n-процесс: проводит экзамен в Telegram (текст/голос), сверяет ответы с эталонами по ключам, рассчитывает предварительную оценку с аргументацией и надежно сохраняет результаты при многопользовательской работе.

---

## 2. Сценарий работы по одной сцене

Студент запускает `/start`, выбирает язык, отправляет регистрационные данные и ответы по ключам билета (текстом или голосом).

Система:
- транскрибирует голос (если требуется)
- сопоставляет ответы с эталонами
- рассчитывает оценки по вопросам и итоговую оценку по билету
- отправляет предварительный результат
- фиксирует попытку в хранилище

---

## 3. Предлагаемый стек

- **Backend:** Python 3.12 + FastAPI (async)
- **Бот:** Telegram Bot API
- **ИИ:**
  - OpenAI (оценивание)
  - Whisper/STT (распознавание речи)
- **Оценка:**
  - правила полноты ответа
  - sentence-transformers
  - cosine similarity
- **Данные:**
  - PostgreSQL — основное хранилище
  - Google Sheets — операционный слой
- **Фоновые задачи:**
  - Redis + Celery/RQ

---

## 4. Инфраструктурные предположения

- Вход через HTTPS webhook от Telegram
- Стабильный доступ к OpenAI, Google Sheets, Telegram
- Изоляция сессий пользователей
- Поддержка многопользовательской работы
- Идемпотентная запись результатов
- Полное логирование сценария экзамена

---

## 5. Ограничения

- Запрещено раскрывать эталонные ответы
- Лимит экзамена: 2 часа с момента `/start`
- Поддержка языков: RU / KK / EN
- Ответ может быть на любом языке
- Оценка возвращается:
  - на русском
  - и на языке ответа
- Оценка — предварительная

---

## 6. Правила дальнейших изменений

- Не ломать структуру результатов
- Версионировать алгоритмы оценки
- Не использовать хардкод (multi-tenant конфигурация)
- Запись результатов:
  - атомарная
  - идемпотентная
  - с retry
- Развитие по этапам:
  1. базовый функционал
  2. multi-bot
  3. аренда/оплата

---

## 7. Первый этап реализации (MVP)

**Срок: 2–3 недели**

- FastAPI webhook + FSM
- Поддержка text + voice
- Транскрибация в единый текстовый поток
- Сопоставление "ключ → эталон"
- Базовая оценка
- Запись в PostgreSQL
- Синхронизация в Google Sheets
- 1 бот, 1 дисциплина, 3 языка

---

## 8. Модель дисциплин (Multi-discipline)

Система должна поддерживать 3–5 дисциплин одновременно.

Каждая дисциплина:
- связана с отдельной Google Sheets таблицей

Структура таблицы:

- reference_answers — эталоны
- student_answers — ответы студентов
- metadata — версия и конфигурация

Google Sheets используются как:
- интерфейс кафедры
- экспортный слой

❗ Данные не удаляются

Используются поля:
- group_id
- session_id
- attempt_id

---

## 9. Модель групп и сессий

- Одна дисциплина → несколько групп
- В группе ~12 студентов
- Одновременно 3–6 активных

Система должна обеспечивать:
- параллельные сессии
- изоляцию пользователей
- группировку результатов

---

## 10. Стратегия бота

- Один бот обслуживает:
  - несколько дисциплин
  - несколько групп

Маршрутизация по:
- discipline_id
- group_id
- session_id

❗ Отдельный бот на группу не требуется

---

## 11. Коммерциализация

- Дисциплины могут быть платными
- Требуется:
  - признак monetized
  - управление доступом
  - возможность расширения до биллинга

---

## 12. Нагрузочные предположения

- 3–5 дисциплин
- 3–6 пользователей одновременно на дисциплину
- Пик: ~20–25 пользователей

Система должна быть:
- асинхронной
- неблокирующей
- устойчивой к API-задержкам

---

## 13. Целостность данных

- Потери данных недопустимы
- Все записи:
  - атомарны
  - идемпотентны
  - с retry

- Обязателен аудит:
  - история
  - версии
  - трассировка
